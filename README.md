## 🔄 Guía de actualización semanal

Esta guía explica cómo reemplazar el archivo `index.html` cada semana para que la página web pública se actualice automáticamente.

### ✅ Antes de empezar

Asegúrate de tener:

- [ ] El nuevo archivo HTML con la información actualizada **guardado en tu computadora**
- [ ] El archivo **renombrado como `index.html`** (¡este nombre es obligatorio!)
- [ ] Tu usuario y contraseña de GitHub a la mano
- [ ] Una conexión a internet estable (el archivo pesa ~10 MB)

---

### 📝 Paso a paso

#### 1. Entrar al repositorio

Abre tu navegador y entra a:

👉 https://github.com/ZetabeAI/Media_Efficiency_Planner

Inicia sesión si te lo pide.

#### 2. Eliminar la versión anterior del HTML

1. En la lista de archivos del repo, haz **clic sobre `index.html`**
2. Arriba a la derecha del archivo, busca el ícono de **basurero** 🗑️ y haz clic
3. Te aparecerá una pantalla de confirmación. Abajo, en el cuadro **"Commit changes"**:
   - Mensaje sugerido: `Eliminar versión anterior - semana DD/MM/AAAA`
4. Haz clic en el botón verde **"Commit changes"**

#### 3. Subir el nuevo HTML

1. Regresa a la página principal del repo (clic en el nombre `Media_Efficiency_Planner` arriba)
2. Haz clic en el botón **"Add file"** → selecciona **"Upload files"**
3. **Arrastra el nuevo `index.html`** desde tu computadora a la zona de carga
   - O haz clic en **"choose your files"** y selecciónalo
4. Espera a que cargue al 100% (puede tardar 30-60 segundos por el peso)
5. Abajo, en el cuadro **"Commit changes"**:
   - Mensaje sugerido: `Actualización semanal - DD/MM/AAAA`
6. Haz clic en el botón verde **"Commit changes"**

#### 4. Verificar la actualización

1. Espera **1 a 3 minutos** (GitHub tarda en propagar los cambios)
2. Abre la página pública: https://zetabeai.github.io/Media_Efficiency_Planner/
3. **Importante:** presiona `Ctrl + F5` (Windows) o `Cmd + Shift + R` (Mac) para forzar refresco y limpiar caché
4. Confirma que la información se vea actualizada ✅

## ⚠️ Reglas importantes

| Regla | Por qué |
|-------|---------|
| El archivo **SIEMPRE** debe llamarse `index.html` | Así GitHub Pages lo carga automáticamente |
| **No** edites el HTML directamente desde la web de GitHub | El archivo es muy grande (10+ MB) y el editor web no funciona |
| **No** borres el repositorio | Se pierde el historial y la URL pública |
| **No** cambies la configuración de GitHub Pages | Se rompe la URL pública |
| **No** cambies la visibilidad de Public a Private | GitHub Pages dejaría de funcionar |

## 🆘 Solución de problemas comunes

### Subí el archivo pero no veo cambios en la página
- Espera 2-3 minutos más, a veces GitHub tarda en propagar
- Fuerza el refresco con `Ctrl + F5` o `Cmd + Shift + R`
- Verifica en una ventana de incógnito del navegador

### Subí el archivo con otro nombre por error
- Borra el archivo mal nombrado (clic en él → basurero 🗑️)
- Renombra el archivo en tu computadora a `index.html`
- Súbelo de nuevo

### Necesito volver a una versión anterior (rollback)
1. En el repo, haz clic en **"Commits"** (arriba a la izquierda, junto al ícono de reloj)
2. Busca el commit de la semana a la que quieres regresar
3. Haz clic en el ícono `<>` (Browse files) de ese commit
4. Clic en `index.html` → botón **"Raw"** → guarda el archivo
5. Súbelo nuevamente siguiendo el paso 3 de la guía

### La carga del archivo se queda colgada
- Verifica tu conexión a internet
- Cierra la pestaña y vuelve a intentar
- Si pesa más de 25 MB, GitHub lo rechaza — revisa que sea el archivo correcto
